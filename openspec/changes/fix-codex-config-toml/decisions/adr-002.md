# ADR-002: Publish the Codex catalog in the documented repository location

- Status: proposed
- Change: fix-codex-config-toml
- Date: 2026-07-09

## Context

The release tree currently writes a root Codex manifest while retaining a Claude-specific marketplace. Current Codex guidance defines repository catalogs at `.agents/plugins/marketplace.json` with structured plugin metadata.

## Decision

Publish `.agents/plugins/marketplace.json` and an isolated payload at `plugins/codex/ospec-workflow`, retaining Claude paths unchanged.

## Alternatives

- Root `marketplace.json`: not the documented repository catalog path.
- Share Claude manifest/payload: incompatible schemas and collision risk.

## Consequences

The release artifact has two target-specific marketplaces and sparse checkout needs both catalog and payload paths. The path change is reversible through republishing the release branch.
