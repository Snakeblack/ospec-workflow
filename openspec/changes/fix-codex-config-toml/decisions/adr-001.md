# ADR-001: Use documented Codex Git marketplace registration

- Status: proposed
- Change: fix-codex-config-toml
- Date: 2026-07-09

## Context

The release implementation uses a Git fragment and `codex plugin add`, neither of which appears in current official Codex plugin documentation. The released end-user procedure is a public compatibility contract.

## Decision

Register `snakeblack/ospec-workflow` with `--ref release` and sparse catalog/payload paths; direct users to install from `/plugins`.

## Alternatives

- Git URL `#release` plus `plugin add`: undocumented current syntax.
- Local clone only: fails the remote-release distribution goal.

## Consequences

The command follows current documentation but installation remains interactive unless a documented noninteractive command emerges. This is easily revised if OpenAI adds one.
