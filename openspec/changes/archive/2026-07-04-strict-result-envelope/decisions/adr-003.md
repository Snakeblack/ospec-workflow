# ADR-003: Shared dependency-free validator mirrored in Go

- Status: proposed
- Change: strict-result-envelope
- Date: 2026-07-03

## Context

The fenced envelope must be validated identically by the JS hook, the Go mirror hook, and
(as primary source) the orchestrator. The repo constraint is Node-22 CommonJS with no external
deps and a byte-for-byte Go mirror (E1 pattern).

## Decision

A standalone `scripts/lib/result-envelope.js` exports `extractEnvelope` and `validateEnvelope →
{valid, errors}` (never throws), mirrored by a new `internal/resultenvelope` Go package. The one
canonical schema is the existing §D field table + Assumption Entry + Blocking Question schemas —
referenced, not redefined. Shared golden fixtures under `internal/testdata/parity/` enforce
parity for the second hook.

## Alternatives

- JSON Schema + ajv — external dependency, violates the dep-free constraint.
- Inline validation inside the hook — not reusable by the orchestrator, harder to unit-test and
  to mirror in Go.

## Consequences

Easier: independently testable, reusable, clean parity boundary. Harder: two implementations to
keep in lockstep (mitigated by shared fixtures + the parity contract). Reversible: delete the
module and the fixtures; the hook degrades to `skill_resolution`-only.
