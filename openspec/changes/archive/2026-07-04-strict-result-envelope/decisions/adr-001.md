# ADR-001: Envelope emission as a `json:result-envelope` fenced block

- Status: proposed
- Change: strict-result-envelope
- Date: 2026-07-03

## Context

The §D Result Contract is "JSON inside markdown by convention"; the orchestrator parses it with
the LLM and `SubagentStop` cannot reliably locate structured fields. We need an unambiguous,
machine-parseable anchor without removing the human-readable prose.

## Decision

Every phase appends exactly one fenced block with info-string `json:result-envelope` carrying the
§D fields as strict, directly `JSON.parse`-able JSON. Emission is additive: `executive_summary`
stays prose. Optional fields are omitted (never `null`).

## Alternatives

- Replace prose with JSON — humans lose readability; spec forbids it.
- Bare ```` ```json ```` fence — not disambiguable from other JSON in the return.
- HTML-comment sentinel — non-standard fence, harder to validate.

## Consequences

Easier: cheap regex extraction shared by hook and orchestrator, no LLM re-interpretation.
Harder: phases must emit valid JSON (mitigated — validator + fail-safe degrade). Reversible:
remove the emission instruction and the consumers fall back to prose parsing.
