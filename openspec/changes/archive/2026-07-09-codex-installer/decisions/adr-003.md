# ADR-003: Ship Codex config as a generator-managed source root

- Status: proposed
- Change: codex-installer
- Date: 2026-07-09

## Context

The clarified spec requires the installer to source managed config values from the generated source-tree `.codex/config.toml`, but the current generator source roots do not include that file.

## Decision

Add source `.codex/config.toml` and include it in `SOURCE_ROOTS` so `runConfigure({ target: "codex" })` emits `dist/codex/.codex/config.toml` alongside generated agents.

## Alternatives

- Generate config only in the installer — hides managed values from generator review.
- Store config under `.codex-plugin` — conflates plugin bundle with separate Codex runtime config.
- Infer limits from agent files — invents behavior beyond the source config.

## Consequences

Managed config becomes reviewable, fixture-backed, and available to both global and repo-local install flows. This slightly expands the generator's source roots, so golden and real-repo tests must guard against unintended target leakage.
