# ADR-002: Text-preserving constrained TOML merge

- Status: proposed
- Change: codex-installer
- Date: 2026-07-09

## Context

The installer must update `.codex/config.toml` without clobbering unrelated user settings, comments, or formatting. The managed key set is limited to `[agents]` and `skills.config` entries present in the generated source config.

## Decision

Implement a dependency-free text patcher that extracts only managed `[agents]` assignments and `skills.config` from `dist/codex/.codex/config.toml`, then inserts or replaces only those assignments in the destination config.

## Alternatives

- Add a TOML parser dependency — conflicts with the current no external runtime dependency pattern.
- Re-serialize the whole TOML file — risks deleting comments and user formatting.
- Hard-code managed defaults in installer code — violates the clarified source-of-truth contract.

## Consequences

User config preservation is testable and rollback is limited to managed lines. The patcher must stay intentionally narrow; future managed keys require updating the source config and tests rather than broadening installer inference.
