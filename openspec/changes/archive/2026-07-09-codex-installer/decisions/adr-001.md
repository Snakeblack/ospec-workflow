# ADR-001: Marketplace wrapper for Codex plugin registration

- Status: proposed
- Change: codex-installer
- Date: 2026-07-09

## Context

Codex install must register the plugin when the CLI is available, but Codex plugin installation uses marketplace commands rather than consuming the generated plugin tree directly. Setup must also remain successful when the CLI is unavailable.

## Decision

Generate a local `dist/codex-marketplace` wrapper containing `plugins/ospec-workflow/`, register it with `codex plugin marketplace add <absolute path>`, then install/update with `codex plugin add ospec-workflow@ospec-tools`; when no `codex` binary is found, print manual commands and exit successfully after build.

## Alternatives

- Register `dist/codex` directly — does not match marketplace command shape.
- Edit `~/.codex/config.toml` marketplace tables directly — more brittle and bypasses the CLI contract.
- Fail when CLI is absent — contradicts REQ-install-001 fallback behavior.

## Consequences

Automatic installs follow Codex CLI semantics and remain recoverable through printed commands. The wrapper adds one generated dist directory, but rollback is simple because generated artifacts are disposable and CLI registration is user-local.
