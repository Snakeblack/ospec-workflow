# ADR-001: Minimal inline wiring + §3.2-sourced commands↔agents contract test

- Status: proposed
- Change: wire-sdd-document
- Date: 2026-07-05

## Context

The orchestrator body is size-guarded under 500 lines (currently 491). `sdd-document` must
be wired without inlining its route protocol, and the J1 defect class (a command routes to an
agent absent from its router's allowlist) must be caught mechanically. Command frontmatter
`agent:` is always `sdd-orchestrator`, so the real routing target lives elsewhere.

## Decision

Wire `sdd-document` with only three inline rows (frontmatter `agents:` entry — in-place, 0 net
lines; one command-index bullet; one pointer-table row → 493 lines) and place the full
protocol in `skills/_shared/route-document.md` per the §15 handler pattern. Add a static test
that parses the §3.2 "Command Roster" table, extracts each row's post-`→` routing target, and
asserts it exists in the router's `agents:` allowlist; rows with no `→` are skipped.

## Alternatives

- Inline the route protocol in CORE — rejected: violates §15 and breaks the 500-line guard.
- Parse command bodies heuristically for agent mentions — rejected: brittle; the clarification
  fixes §3.2 as the single source of truth.

## Consequences

Easier: future wiring gaps of this shape fail `npm test` automatically. Harder: the §3.2 table
becomes a load-bearing contract that must stay in sync. Reversible: revert the rows, handler,
and test to restore the prior state.
