# ADR-002: Orchestrator-owned, git-scoped post-run sandbox verification (J5)

- Status: proposed
- Change: wire-sdd-document
- Date: 2026-07-05

## Context

`sdd-document` declares a write sandbox (its output dir plus `/AGENTS.md`, `/CLAUDE.md`), but a
self-reported sandbox claim is not evidence. An unscoped `git status` would false-positive on
pre-existing untracked paths (e.g. the current untracked `openwiki/`). Option C custom paths may
resolve outside the repo, where `git` cannot observe writes.

## Decision

The orchestrator independently resolves the approved output dir from the `scope_choice`/
`custom_path` it passed at launch (A→`openwiki/`, B→`docs/wiki/`, C→the same `custom_path`),
rejects any Option C path resolving outside the repo root at gate time, and after the executor
returns `success` runs a `git status` scoped to `{output_dir, /AGENTS.md, /CLAUDE.md}`. Any path
outside that set halts with a two-option `question_gate`: "Abort and leave files for manual
review" (default/recommended) vs "Acknowledge and close anyway (accepted risk)". The route never
closes without an explicit choice; the executor's success report cannot override the halt.

## Alternatives

- Trust the executor's completion report — rejected: reinstates the "declared contract, no
  verification" defect class this change generalizes.
- Unscoped `git status` — rejected: false-positives on unrelated pre-existing untracked paths.

## Consequences

Easier: sandbox escapes are caught authoritatively and independently. Harder: the orchestrator
must compute and reject out-of-repo custom paths before delegating. Reversible via revert.
