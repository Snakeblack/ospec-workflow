# ADR-003: Batched single gate + `.last-update.json` schema extension (J4)

- Status: proposed
- Change: wire-sdd-document
- Date: 2026-07-05

## Context

Language and scope were two separate blocking round-trips, and `.last-update.json` carried
neither, so update runs could not skip re-asking. Update mode also had no user-facing surface to
force a re-ask (no flag/argument-hint exists).

## Decision

Present language + scope as ONE `question_gate` (two independent questions) via a single
`vscode/askQuestions` call. Extend the `.last-update.json` metadata schema with `doc_language`
and `scope_choice` reflecting the resolved run values. In update mode, when both persisted fields
exist, skip the gate; gate the skip behind a yes/no keep/change pre-question — answering "change"
acts as the explicit override and re-asks only the selected field(s).

## Alternatives

- Keep two sequential gates — rejected: two blocking round-trips for one decision point.
- Persist answers only in `state.yaml` — rejected: `.last-update.json` is the wiki's own durable
  metadata that update runs already read to scope the diff window.

## Consequences

Easier: subsequent update runs are non-interactive when metadata is complete. Harder: the
`.last-update.json` schema is now a compatibility contract (older files without the fields fall
back to init-mode gating). Reversible via revert.
