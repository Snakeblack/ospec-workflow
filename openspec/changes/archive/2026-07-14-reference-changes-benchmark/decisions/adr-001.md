# ADR-001: Separate infrastructure delivery from live baseline operation

- Status: proposed
- Change: reference-changes-benchmark
- Date: 2026-07-14

## Context

O2 must deliver a trustworthy benchmark mechanism, while remote model availability, nondeterminism and quota make a live baseline operationally variable. Treating those external conditions as implementation acceptance would keep a locally verifiable capability open indefinitely.

## Decision

The catalog, runner, guards, run-level scorer, model-plus-reasoning-effort identity, installed-byte identity, cache, offline recovery and atomic publisher form the archiveable deliverable. Their local tests and structural checks determine verification; neither a live core run nor an existing baseline is required to close the change.

A later operational run may publish `reference-baseline.md` only from three accepted core rows with exactly compatible identities. Partial sets fail closed. The accepted Sol observation and rejected Luna-low observation remain diagnostics outside comparable baseline completion; any retained cache identity stays isolated and neither observation counts toward 3/3. O1 remains supplementary, and integrity claims remain limited to the cooperative-orchestrator threat model.

This ADR remains `proposed` throughout the active change and becomes final only through archive.

## Alternatives

- Block archive on live 3/3: rejected because quota and remote behavior are not infrastructure defects.
- Publish partial or mixed observations: rejected because the result would not be comparable.
- Require all nine profiles: rejected because `extended` is an optional diagnostic suite.
- Drop live publication entirely: rejected because the verified mechanism should remain operationally usable.

## Consequences

O2 can close on deterministic evidence without weakening publication integrity. Operators must schedule and fund the live follow-up separately, and no baseline exists until comparable 3/3 evidence is available. Diagnostics stay auditable but cannot be promoted. The separation is reversible by a future spec change, not by bypassing current fail-closed guards.
