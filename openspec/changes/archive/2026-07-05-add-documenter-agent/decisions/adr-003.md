# ADR-003: Interactive Launch Gate for Scope Selection

- Status: proposed
- Change: add-documenter-agent
- Date: 2026-07-05

## Context
The documentation agent needs a way to determine the output path and scope of the technical wiki before running. We must allow the user to select the appropriate scope dynamically without adding hardcoded defaults or complex CLI arguments.

## Decision
Implement a launch-blocking `question_gate` payload in the `sdd-document` agent startup flow. The gate prompts the user to select Option A, B, or C, and validates the path at resuming when Option C is selected, blocking again on fuzzy/invalid paths.

## Alternatives
- Hardcode the output path to a single directory: Rejected because it does not support multiple wiki styles or user configurations.
- Prompt using standard stdin CLI prompts: Rejected because it violates the declarative agent execution model and bypasses orchestrator gate handling.

## Consequences
- Easy: Enables flexible, structured selection of documentation scopes.
- Hard: Requires the orchestrator to handle the blocking state and display the `question_gate` options to the user. Easily reversible by updating the launch logic.
