# ADR-001: sdd-document as a Dedicated Executor Agent

- Status: proposed
- Change: add-documenter-agent
- Date: 2026-07-05

## Context

The repository requires a way to compile specs, configuration, and active change metadata into wiki pages automatically. We need to decide where this logic lives within the agent architecture without violating tool or routing boundaries.

## Decision

Introduce `sdd-document` as a dedicated phase executor agent (`agents/sdd-document.agent.md`) that is non-user-invocable (`user-invocable: false`), registers the `/sdd-document` slash command routing via the orchestrator, and uses the `default` tier in `models.yaml`.

## Alternatives

- Implement logic inline in `sdd-orchestrator`: Rejected because the orchestrator is a coordinator, and execution of compilation violates the executor boundary.
- Implement as a standalone CLI tool script: Rejected because compiling status, drift, and specs requires LLM context synthesis.

## Consequences

- Easy: Enforces the coordinator-executor split and uses standard routing.
- Hard: Requires registering the command and agent configuration in the catalog and model routing. Reversible by removing agent and command registration.
