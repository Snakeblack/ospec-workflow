# ADR-003: Dual-directory write sandbox modeled as an approved SET

- Status: proposed
- Change: starlight-web-doc
- Date: 2026-07-06

## Context
Options A/B/C approve a single output directory. Option D writes to two sibling directories (`openwiki/` and `web-doc/`) in one run, and the orchestrator-owned post-run J5 inventory (agents REQ-006) must verify both without false positives.

## Decision
Model the approved output as a SET. For scope D the set is `{openwiki/, web-doc/}`; the executor may write to either and halts (`design-mismatch`) on any write outside both minus the `/AGENTS.md`,`/CLAUDE.md` exception. Route §3 resolves the set and §6 J5 scopes `git status` to both dirs; the orchestrator resolves the set authoritatively from `scope_choice`, not from the executor's self-report.

## Alternatives
- Treat `web-doc/` as nested under one root — rejected: the dirs are siblings, not nested.
- A second independent J5 pass — rejected: duplicates the check; one scoped pass over the set suffices.

## Consequences
Generalizes the existing single-dir contract with minimal churn; A/B/C keep a one-element set. Slightly more complex J5 scoping. Reversible with the rest of Option D.
