# Starlight Sidebar, Theming & i18n Reference

Source: starlight.astro.build (guides/sidebar, guides/customization, guides/overriding-components). Verified 2026-07.

## Sidebar

Item kinds inside `starlight({ sidebar: [...] })`:

```js
sidebar: [
	// Internal link — page title used as label
	'constellations/andromeda',           // shorthand
	{ slug: 'constellations/orion' },     // object form
	{ label: 'Home', link: '/' },         // explicit label + link (also external URLs)

	// Group (nestable)
	{
		label: 'Constellations',
		collapsed: true,                    // collapsed by default
		items: [
			'constellations/carina',
			{ label: 'Seasonal', items: ['constellations/andromeda'] },
		],
	},

	// Autogenerated from a docs directory (alphabetical by file id)
	{
		label: 'Reference',
		items: [{ autogenerate: { directory: 'reference', collapsed: true } }],
	},
],
```

Extras per item:

```js
// Badge — variants: note | tip | danger | caution | success
{ slug: 'stars/sirius', badge: { text: 'Stub', variant: 'caution' } }

// Custom HTML attributes
{ label: 'NASA', link: 'https://www.nasa.gov/', attrs: { target: '_blank' } }

// Translated labels (BCP-47 keys)
{ label: 'Andromeda', translations: { 'pt-BR': 'Andrômeda' }, slug: '…' }
```

Per-page control from frontmatter (`sidebar:` field): `label`, `order` (lower = higher), `hidden: true`, `badge`, `attrs`. Autogenerated entries merge `autogenerate.attrs` with page-level `attrs`.

## Theming & CSS

Load custom CSS via `customCss: ['./src/styles/custom.css']` and override the `--sl-*` custom properties:

```css
/* src/styles/custom.css */
:root {
	--sl-font: 'Roboto', sans-serif;      /* after @font-face or Fontsource import */
	--sl-color-accent: hsl(234, 90%, 60%);
}
```

- Theme-aware logo: `logo: { light: '…', dark: '…', replacesTitle: true }`.
- Wide landing layout: `template: splash` frontmatter.
- Global scripts/meta: `head` config option; per-page via `head` frontmatter.

## Overriding UI Components

Map component names to replacements in config; access route data through `Astro.locals.starlightRoute`:

```js
starlight({
	components: { PageTitle: './src/components/Title.astro' },
});
```

```astro
---
// src/components/Title.astro
const { title } = Astro.locals.starlightRoute.entry.data;
---
<h1 id="_top">{title}</h1>
```

Prefer overrides + custom CSS over copying theme internals — forks break on upgrade (`npx @astrojs/upgrade`).

## Internationalization

```js
starlight({
	defaultLocale: 'en',
	locales: {
		en: { label: 'English' },
		'zh-cn': { label: '简体中文', lang: 'zh-CN' },
		ar: { label: 'العربية', dir: 'rtl' },
	},
});
```

- Content mirrors locale dirs: `src/content/docs/en/…`, `src/content/docs/zh-cn/…`.
- Untranslated pages automatically fall back to `defaultLocale` content with a notice.
- `title` accepts `Record<locale, string>`; sidebar labels/badges accept `translations`.
